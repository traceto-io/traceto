# Traceto API Reference

Base URL: `https://ingest.traceto.io` (production) · `http://localhost:8000` (local)

All requests require the `X-Api-Key` header with a valid `tr_` prefixed key.

---

## Authentication

| Header | Value | Required |
|---|---|---|
| `X-Api-Key` | `tr_<your_key>` | All endpoints |

Errors: `401` invalid/inactive key · `402` monthly quota exceeded.

---

## POST /v1/captures

Ingest one or more captured HTTP interactions.

**Rate limit:** 2 000 requests/minute per IP.

### Request body

```json
{
  "captures": [
    {
      "service": "my-api",
      "session_id": "abc123",
      "request": {
        "method": "POST",
        "path": "/orders",
        "query_params": {},
        "headers": { "content-type": "application/json" },
        "body": { "item_id": 42, "quantity": 1 }
      },
      "response": {
        "status_code": 201,
        "headers": { "content-type": "application/json" },
        "body": { "order_id": 99, "status": "pending" },
        "latency_ms": 34.2
      }
    }
  ]
}
```

| Field | Type | Required | Notes |
|---|---|---|---|
| `captures` | array | ✅ | 1–500 items per call |
| `service` | string | ✅ | Logical service name (matches `traceto.config.yaml`) |
| `session_id` | string | — | Used for chain analysis |
| `request.method` | string | ✅ | `GET`, `POST`, etc. |
| `request.path` | string | ✅ | URL path without query string |
| `request.body` | any | — | JSON-serialisable; PII is scrubbed server-side |
| `response.status_code` | int | ✅ | HTTP status |
| `response.latency_ms` | float | — | End-to-end latency |

### Response `202 Accepted`

```json
{ "accepted": 3 }
```

### Errors

| Code | Meaning |
|---|---|
| `401` | Missing or invalid API key |
| `402` | Monthly quota exceeded |
| `413` | Request body > 10 MB |
| `429` | Rate limit exceeded |

## POST /v1/generate-tests

Generate pytest files from all captures for a service.

### Query parameters

| Param | Required | Description |
|---|---|---|
| `service` | ✅ | Service name to generate tests for |

### Response `200`

```json
{
  "generated": 4,
  "files": [
    { "file": "test_post_orders.py", "test_count": 3 }
  ],
  "code": {
    "test_post_orders.py": "# Generated by Traceto..."
  }
}
```

Returns `{ "tests": [], "message": "No captures found" }` when no data exists yet.

## GET /v1/coverage

Return per-endpoint capture statistics.

### Query parameters

| Param | Required | Description |
|---|---|---|
| `service` | ✅ | Service name |

### Response `200`

```json
{
  "service": "my-api",
  "total_captures": 142,
  "endpoints": [
    {
      "method": "POST",
      "path": "/orders",
      "captures": 38,
      "statuses": [200, 201, 422]
    }
  ]
}
```

## GET /v1/billing/usage

Return current month's quota usage for your API key.

### Response `200`

```json
{
  "plan": "starter",
  "month": "2026-04",
  "used": 48210,
  "quota": 1000000,
  "remaining": 951790,
  "percent_used": 4.8
}
```

## GET /health

Health check — no auth required.

```json
{ "status": "ok", "version": "0.1.0" }
```

## Stripe Webhook — POST /v1/billing/webhook

Internal endpoint for Stripe events. Configure in the Stripe Dashboard:

- **URL:** `https://your-domain.com/v1/billing/webhook`
- **Events:** `customer.subscription.updated`, `customer.subscription.deleted`, `invoice.payment_failed`
- Requires `STRIPE_WEBHOOK_SECRET` env var for signature verification.
